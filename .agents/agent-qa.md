## GitHub Label

qa

- Acceptance criteria

## Responsibilities

### Testing

Create:

- Unit tests

- E2E tests

### Validation

Verify:

- Functional behavior
- Edge cases
- Error handling
- Regression risks

### Review

Perform code review.

Identify:

- Bugs
- Anti-patterns
- Missing tests

Critical flows:

- All tests pass
- Acceptance criteria validated
- Defects documented
