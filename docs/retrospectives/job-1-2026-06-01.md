## What we wanted

End-to-end autonomous squad: HF/Actions agents, minimal Director input (approve BA + release only), visible pipeline, shippable VS Code extension.

## What happened

| Area | Outcome |
|------|---------|
| Developer | Target PR merged; extension scaffold in `vscode-squad-director` |
| Validation | Sub-issues #80–#84 opened; inconsistent HF/Actions results; parent advanced to `validation` without reliable reports |
| Director visibility | Web dashboard iterated (4 columns → agent tables → rate limits → “approvals only”) and **hid stuck state** while #64 sat in `validation` |
| Orchestration | `advance-implemented` ↔ `phase-tick` **infinite loop**; validation re-dispatch skipped; wrong HF model for tech-writer |
| Director UX | “Nothing needs your approval” while job was **stuck** — true for gates, false for pipeline health |

## Root causes

1. **Mixed signals** — “Director dashboard” tried to be both approval inbox and ops center; when simplified to approvals-only, it lied by omission.
2. **GitHub API from laptop** — live `gh` on every refresh hit rate limits; snapshots were stale.
3. **Orchestration bugs** — recursive phase tick; validation dispatch not re-run; closed issues classified as blocked.
4. **Agent context** — Release Manager reported “missing BA” when BA existed; stub HF test reports counted as deliverables.
5. **No single source of truth** — Issues, Actions, dashboard JSON, and Project board disagreed.

## What to keep

- Issue-first deliverables on `ai-alpha-squad`; code on target repo.
- `SQUAD_ORCHESTRATOR_TOKEN` + HF/Actions dispatch in CI.
- `squad-director-now.sh` as the Director CLI (approvals + attention).
- Phase tick / phase watch on GitHub (not local machine).
